Project:
Software Issue and Change Management

Project Description

This project required building a repository for data about changes and their corresponding issues. The user of this repository will introduce data manually via a web based user interface. They should also be able to run reports about the collected data and simple analysis (such as, finding parts of the code that were changed for multiple issues, or retrieving only the issue title). The system should also check for inconsistencies automatically. Some data will be entered in forms of links to repositories or issues in the issue tracker system, in which case the software should collect automatically the data from that URL.
